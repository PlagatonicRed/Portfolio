Basic 3D renderer

A basic 3D renderer built from scratch using SDL2. Click here or on the cover image to visit the project repository.

Development Details

  • Development Time: 2 weeks
  • Team Size: Solo
  • Goals:
    • Be able to apply linear algebra concepts to a digital project
    • Be able to deepen understanding of 3D graphics and 3D transformations

The hardest part of this project was teaching myself the necessary linear algebra concepts to carry out 3D rendering. At the time, I had not taken any courses or had much prior knowledge of linear algebra topics. Specifically, I had to learn how to use matrix transformations to manipulate 3D vectors, enabling me to mathematically simulate both 3D translation and 3D rotation of points.

While the basics of the linear algebra concepts related to rendering weren't too difficult to grasp, integrating a camera system and understanding the associated matrix math was far more complex. Wrapping my head around these concepts took up a significant portion of the development time.

Though, while the basics of the linear algebra concepts relating to rendering weren't that hard to grasp, integrating a camera system and the math matrix math that it involves is much more convuluted and wrapping my head around this took up a large portion of development time.

I've optimized the renderer by implementing vertex culling to discard vertices not visible from the camera's perspective and added basic shading. The shading leverages the dot product between the light direction vector and the normal of each triangle to create a shadow gradient that imitates lighting.

Conway's Life

A basic digital drawing program with Conway's game of life integrated into it. Click here or on the cover image to visit the project repository.

Development Details

  • Development Time: 2 weeks
  • Team Size: Solo
  • Goals:
    • Practice using the SDL2 library within an application
    • Create A personal digital drawing program for private use

This project originally started as a simple program to simulate Conway's Game of Life. However, during development, I realized one of my debugging tools inadvertently created a basic drawing program.

The program allows the refresh rate to be manually adjusted using the number keys, controlling both the speed of the Conway simulation and the smoothness of the drawing process. I implemented a line-fill technique to prevent skipped pixels when drawing at lower refresh rates. However, this can result in jagged lines if the refresh rate is set too low.

To optimize the Conway simulation, I used bitwise operations to enable faster computation. Additionally, I plan to incorporate a stack-based undo and redo solution for easier mistake correction in the future.

Text Document Service Backend

A simple containerized program that leverages SQL to allow users to view and modify text documents. Click here or on the cover image to visit the project repository.

Development Details

  • Development Time: 3 weeks
  • Team Size: Solo
  • Goals:
    • Work with databases in order to create a usable backend service
    • Leverage containerization to lower application size

This application consists of four microservices that work together to manage user text document functionalities. The microservices are as follows: the user service, the document manager service, the document search service, and the logging service. The user service manages user-related operations such as account creation, login, and validation. The document manager service handles tasks like creating, editing, and updating documents. The document search service allows users to search for documents based on specific criteria, ensuring efficient retrieval. The logging service tracks user actions and maintains a history of their activities, giving users access to a detailed record of their interactions within the system.

Each microservice is containerized using a dedicated Docker image, ensuring that each system operates in complete isolation from the others while maintaining full functionality as an independent unit. This approach encapsulates all the necessary dependencies, configurations, and runtime environments within the container, making each microservice self-sufficient and portable. Containerization also enables seamless communication between the services, allowing them to work together as a cohesive system while remaining decoupled. This isolation enhances reliability, simplifies debugging, and ensures that changes or issues within one microservice do not impact the performance or stability of the others.
